# CLAUDE.md - правило для Claude Code / Dev A

Этот репозиторий содержит текущий live-фронт сайта САНСЛ. Текущий фронт
считается продуктовым эталоном и не должен быть переписан, перегенерирован или
визуально "улучшен" без прямого задания founder.

## Главная инструкция

Dev A: при дальнейшей работе с сайтом не трогай текущий фронт по умолчанию.
Сохраняй существующую стилистику, структуру, ритм, фон, анимации, типографику и
mobile-first поведение. Любая правка должна быть точечной и объяснимой.

Если задача не требует изменения визуального слоя, не меняй:

- `index.html`;
- папку `assets/`;
- текущий переливающийся canvas-background;
- header с логотипом/кубом САНСЛ;
- композицию mobile-first листов;
- карточки Human Graph Engine;
- экран "Твой LifeOS" и маршрутную линию;
- экран "Три двери";
- экран "Скоро открываем двери";
- текущие изображения, отступы, цвета, тени, анимации и шрифтовую систему.

## Что считается запрещенным без отдельного подтверждения founder

- Полная перегенерация `index.html`.
- Замена текущего визуального языка на другой стиль.
- Удаление переливающегося фона или замена его статичной картинкой.
- Добавление темных/матовых плашек поверх текущего фона, если founder прямо не
  попросил.
- Возврат 3D-головы, кубов, Spline-сцен или других тяжелых 3D-объектов, если
  это не указано в новой задаче явно.
- Массовое форматирование HTML/CSS/JS, из-за которого diff становится
  нечитаемым.
- Изменение текста, дизайна или порядка экранов "заодно".

## Если будущая задача все-таки требует правки фронта

Перед изменениями:

1. Сначала определи точный блок, которого касается задача.
2. Не редактируй соседние блоки.
3. Сделай restorable backup или отдельную git-ветку перед рискованной правкой.
4. Сохрани текущую визуальную систему: мягкий переливающийся фон, чистые белые
   тексты, премиальную минималистичную подачу, крупные mobile-first экраны,
   аккуратные анимации без резких кислотных эффектов.
5. Не добавляй новые библиотеки, тяжелые ассеты или внешние 3D-сцены без
   отдельной причины.

После изменений:

1. Проверь `git diff` и убедись, что изменены только нужные строки.
2. Проверь мобильный viewport в первую очередь.
3. Проверь, что переливающийся фон работает.
4. Проверь, что свайпы/скролл не сломаны.
5. Если случайно изменил текущий фронт вне задачи - остановись и верни
   незапрошенную часть.

## Prompt для начала каждой сессии Dev A

Перед любыми правками повтори себе:

> Я работаю с текущим live-фронтом САНСЛ как с утвержденным продуктовым
> эталоном. Я не переписываю сайт, не меняю стиль и не трогаю существующий
> фронт без прямого задания. Если нужно добавить или исправить что-то, я делаю
> минимальный точечный diff, сохраняю переливающийся фон, mobile-first
> композицию, текущую типографику, цвета, ритм, анимации и премиальный
> минималистичный характер сайта.

## Критерий готовности

Любая будущая работа Dev A считается корректной только если:

- текущий фронт не изменен без прямого задания;
- стиль сайта визуально сохранен;
- мобильная версия не ухудшилась;
- фон остается переливающимся;
- diff узкий и понятный;
- все незапрошенные визуальные изменения отсутствуют.
